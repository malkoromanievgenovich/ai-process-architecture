# WRITING.md — Writing Style, Tone & Content Standards

Words are a sovereign domain — copy, microcopy, UI strings, button and error text, naming, and content. Read this whenever the task touches words.

## Writing Style & Tone

- **Clarity & Truth:** The system must be well-described and informative. Guide the person informatively (using clear, explicit instructions) from point A to point B. Emotional text is noise; plain and accurate is the goal.
- **Objective Phrasing:** Use objective, third-person phrasing. Define value propositions without taking responsibility for individual operator decisions.
- **Concise copy:** Emphasize focus on real value over noise.
- **Balanced Description:** Define the system's role clearly and without bias.
- **Constructive Lexicon:** Use active, constructive terms — "create", "align", "harmonize" — over vague or negatively-framed ones.
- **Affirmative by default:** State the thing in the positive — say what *is*. A negation ("do not…", "немає…") makes the reader build an idea, then discard it: extra cognitive load. Prefer "the choice stays with the person" to "we do not decide"; "say what is" to "don't guess"; "offer, then stop" to "never push". Keep a negation only where it carries real meaning — a genuine limit or a deliberate ban.
- **Don't anthropomorphize the system:** copy and naming never imply the system perceives or thinks. Out: cognition words — "assistant," "smart/intelligent," "analyses," "hints," "suggests," "recommends," "proposes a diagnosis," "thinks"; and perception words — "sees / looks / watches" («дивиться», «подивимось», «дивимось»). In: "enter," "select from the list," "by rule," "required field," "template," "completeness check"; and for the tool's own act, "shows / displays" («показує», «відображає»). Phrase both the decision and the observation as the human's action — the human looks; the tool only displays what is there. A decision is always the human's, never the system's. This also means the system never speaks of itself as a person: no first-person perception/feeling/thought and no "human-like" manner — out «я бачу», «по-людськи», «мені здається», «я відчуваю», «я вважаю». This applies to **every text the system generates** — an audit, a report — not only static UI strings; generated output obeys this lexicon exactly as button text does.
- **Guard generated output at runtime:** the model that writes product text is a third party that follows the prompt loosely — it slips into first-person perception ("спершу подивлюся", "я бачу", "спробувала"). Filter every AI-generated string through this lexicon **before** a person sees it, dropping the parts that pose as a person. The prompt asks; the guard enforces (see `SECURITY.md` — Output Sanitization).
- **No Self-Claimed Virtue:** Copy never asserts the system's own honesty, truth, accuracy, or objectivity — out: "honestly," "truly / really," "accurately," "objectively" («чесно», «насправді», «правдиво», «точно», «об'єктивно»). State what the thing *is* and what it is *built from*, and leave the verdict with the operator.
- **Respectful Address by Default:** The system addresses the operator with respect and never presumes familiarity. Default to the polite/formal register the language provides — Ukrainian «ви», not «ти»; the equivalent T–V or honorific form elsewhere. Informal address is granted by the operator, not assumed by the tool.
